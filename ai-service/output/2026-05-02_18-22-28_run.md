## User Prompt

Design a campus network for an organization with 4 building(s), 3 floor(s) per building. There are approximately 48 students/visitors, 23 staff/faculty, and 3 administrators. Devices needed: laptops, printers, phones, cameras, wifi. Sensitive areas requiring extra security: exams, server. Special roles: Principal, Exam Controller, Hostel Warden. Uptime requirement: important. Expecting growth of 200-500.

## Phase 1: Rephrased Prompt

**Role:** Senior Network Topology Architect
**Task:** Design a scalable, high-availability Campus Area Network (CAN) for an educational organization.

**1. Project Purpose & Scope**
The objective is to design a robust network infrastructure across a campus consisting of 4 buildings, each with 3 floors. The design must support diverse user groups (students, faculty, administration), provide seamless connectivity for various device types, and ensure high availability for critical operations.

**2. User & Device Profiles**
*   **User Population:** 
    *   Initial: ~74 active users (48 students/visitors, 23 staff/faculty, 3 administrators).
    *   Growth Projection: Scalable to accommodate an additional 200-500 users.
*   **Device Requirements:**
    *   **End-points:** Laptops, VoIP phones, Network Printers.
    *   **Infrastructure:** High-density Wireless Access Points (WAPs) for campus-wide coverage and IP Security Cameras for surveillance.
*   **Specialized Role-Based Access Control (RBAC):** Define specific VLANs or access policies for the Principal, Exam Controller, and Hostel Warden.

**3. Performance & Availability Requirements**
*   **Uptime:** High availability is critical. Implement redundancy at the core and distribution layers to eliminate single points of failure (e.g., HSRP/VRRP or StackWise).
*   **Scalability:** The architecture must be modular to support the projected 500+ user growth without requiring a complete forklift upgrade of the core.
*   **Throughput:** Ensure low latency for VoIP and high bandwidth for academic resources and surveillance feeds.

**4. Physical & Logical Topology**
*   **Physical Layout:** 4 Buildings $\times$ 3 Floors.
*   **Topology Recommendation:** Use a Hierarchical Three-Tier Model (Core, Distribution, Access).
*   **Wiring/Connectivity:** Suggest appropriate cabling (e.g., Fiber optic backbones between buildings/floors and Cat6a for horizontal cabling to endpoints).

**5. Security & Segmentation**
*   **Sensitive Zones:** Design isolated, high-security zones for the **Server Room** and **Exam Areas**. Implement strict ACLs and firewalling to prevent unauthorized access.
*   **Network Segmentation:** Create separate VLANs for:
    *   Management (Network gear)
    *   Administrative/Staff (Sensitive data)
    *   Student/Guest (Internet access/Isolated)
    *   IoT/Surveillance (Cameras and Printers)
    *   VoIP (Voice traffic prioritization)

**6. Constraints & Assumptions**
*   **Budget:** Assume a mid-to-high range enterprise budget prioritizing longevity and reliability over the lowest initial cost.
*   **Hardware:** Assume a mix of Layer 3 switches for the core/distribution and Layer 2 PoE+ switches for access floors to power phones and cameras.

**Deliverables Required:**
1.  A detailed logical network diagram.
2.  An IP addressing scheme (IPv4 subnetting).
3.  A hardware bill of materials (BOM) suggestion.
4.  A VLAN mapping table including the special roles requested.

## Phase 2: Network Topology

### 1. Topology Overview
**Type:** Hierarchical Three-Tier Campus Architecture (Core $\rightarrow$ Distribution $\rightarrow$ Access)
**Design Philosophy:** Modular "Building Block" approach. The network utilizes a collapsed core/distribution model per building to minimize latency, linked via a high-speed redundant Core.

---

### 2. Layer Breakdown
*   **Core Layer:** 2x High-Capacity L3 Chassis Switches (Deployed as a Virtual Switch Framework/VSX pair).
*   **Distribution Layer:** 8x L3 Switches (2 per building, deployed in VSF/Stacking pairs).
*   **Access Layer:** 12x L2 PoE+ Switches (1 per floor per building, totaling 12 units).

---

### 3. High-Availability & Traffic Engineering
*   **Control Plane Redundancy:** 
    *   **Core:** VSX (Virtual Switching Extension) for hitless failover and shared control plane.
    *   **Distribution:** VSF (Virtual Switching Framework) to treat two physical switches as one logical entity.
*   **Data Plane Redundancy:** 
    *   **LACP (802.3ad):** All uplinks are Multi-chassis Link Aggregation (M-LAG) bundles.
    *   **Gateway Redundancy:** VRRP (Virtual Router Redundancy Protocol) used at the Distribution layer for subnet gateways.
*   **Quality of Service (QoS):** 
    *   **Strict Priority:** VoIP (EF - Expedited Forwarding).
    *   **Assured Forwarding:** Administrative/Exam traffic.
    *   **Best Effort:** Student/Guest traffic.

---

### 4. Link Design
| Segment | Connectivity | Speed | Redundancy | Cable Type |
| :--- | :--- | :--- | :--- | :--- |
| **Core $\leftrightarrow$ Dist** | 2x 25GbE LAG | 50Gbps | Dual-Homed (Active/Active) | OS2 Single Mode Fiber |
| **Dist $\leftrightarrow$ Access** | 2x 10GbE LAG | 20Gbps | VSF/Stack Link | OM4 Multi-Mode Fiber |
| **Access $\leftrightarrow$ End** | Single Port | 1Gbps | N/A | Cat6a UTP |

### 6. Redundancy & Failover Logic
1.  **L2 Loop Prevention:** MSTP (Multiple Spanning Tree Protocol) is configured, but blocked ports are minimized by using VSX/VSF (Logical Chassis).
2.  **L3 Routing:** OSPF used for routing between Core and Distribution. Equal-Cost Multi-Path (ECMP) ensures load balancing across the 25GbE links.
3.  **Gateway Failover:** VRRP ensures that if one Distribution switch fails, the virtual IP remains active on the peer switch, preventing client disconnects.

## Phase 3: Device Selection & BOM

Based on the approved topology and the retrieved datasheets from the HPE Aruba Networking CX portfolio, here is the detailed Bill of Materials (BoM).

### 1. Layer-by-Layer Analysis & Device Matching

*   **Core Layer:** The requirement specifies a VSX pair of high-capacity L3 chassis switches with 25GbE/50GbE capabilities. The **Aruba CX 6400 Series** (e.g., 6405 v2) is the designated chassis for this role, providing the necessary modularity, VSX support for hitless failover, and high-speed line cards.
*   **Distribution Layer:** The requirement calls for L3 switches in VSF stacking pairs with 10GbE uplinks. The **Aruba CX 6300M Series** is the ideal match. It supports VSF stacking (up to 10 members), provides L3 routing (OSPF, BGP), and features SFP56 ports that auto-negotiate between 1G, 10G, 25G, and 50G, perfectly matching the 25GbE Core $\leftrightarrow$ Dist and 10GbE Dist $\leftrightarrow$ Access link design.
*   **Access Layer:** The requirement is for L2 PoE+ switches (1 per floor). The **Aruba CX 6200F Series** is selected. It provides 48 ports of 1GbE with Class 4 PoE (up to 30W per port), which is sufficient for the laptops, phones, and cameras mentioned. It also includes 4x SFP+ 10GbE ports to support the requested 10GbE LAG uplinks to the distribution layer.

*   **Redundancy:** Core uses **VSX** (active-active), Distribution uses **VSF** (logical chassis), and Access uses **LACP/M-LAG** for dual-homing.
