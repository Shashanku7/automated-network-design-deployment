## Phase 2: Network Topology

As a Senior Network Topology Architect, I have designed the following high-availability infrastructure for the PJ Building. This design utilizes a **Collapsed Core/Distribution Architecture** utilizing HPE Aruba networking standards (VSX/VSF) to ensure zero-single-point-of-failure.

---

### 1. Topology Overview
**Type:** Three-Tier Hierarchical Model (Collapsed Core/Distribution $\rightarrow$ Access).
The architecture follows a "Star-of-Stars" physical layout. The MDF (Main Distribution Frame) resides on the 1st Floor (Admin), with IDFs (Intermediate Distribution Frames) on each subsequent floor.

### 2. Layer Breakdown & Hardware Mapping

#### A. Core/Distribution Layer (MDF - Floor 1)
*   **Role:** High-speed routing, policy enforcement, and inter-VLAN routing.
*   **Configuration:** A pair of high-capacity chassis switches (e.g., Aruba 6400 series) configured in **VSX (Virtual Switching Extension)**.
*   **Function:** Acts as the Default Gateway for all floor VLANs.

#### B. Access Layer (IDFs - Per Floor)
Each floor is treated as a separate access block to contain broadcast domains and localize cabling.

| Floor | Department | Device Density (Total Endpoints) | Access Switch Config | Sizing Logic |
| :--- | :--- | :--- | :--- | :--- |
| **1** | Admin | 66 Endpoints | 1x 48-port PoE+ Switch (VSF Stacked) | Low density; prioritized security/VoIP. |
| **2** | CSE | 400 Endpoints | 3x 48-port PoE+ Switches (VSF Stack) | High density (Students); requires 10G uplinks. |
| **3** | ECE | 238 Endpoints | 2x 48-port PoE+ Switches (VSF Stack) | Medium density; high throughput for labs. |

---

### 3. High Availability & Link Design

*   **Core Redundancy (VSX):** Two switches operating as one logical entity. Unlike stacking, VSX allows for independent control planes, ensuring that a firmware upgrade on one does not crash the entire building.
*   **Access Redundancy (VSF):** Switches on Floor 2 and 3 are grouped in a **Virtual Switching Framework (VSF)** stack to simplify management and provide link redundancy.
*   **Link Aggregation (LACP):** 
    *   **Access to Core:** Each VSF stack is connected to the Core via a **Multi-Chassis Link Aggregation (M-LAG)** bundle.
    *   **Speed:** $2 \times 10\text{GbE}$ SFP+ uplinks per stack (Active/Active) providing 20Gbps throughput.
*   **Failover:** Using **VRRP (Virtual Router Redundancy Protocol)** or VSX-integrated routing to ensure sub-second convergence if a core member fails.

---

### 5. Security & Performance Constraints

#### A. Access Control
*   **802.1X (RADIUS):** Mandatory for all Student/Staff ports. Dynamic VLAN assignment will be used; if a student plugs into an Admin port, they are automatically shifted to the Student VLAN.
*   **Port Security:** MAC-address limiting on Admin/Staff ports to prevent unauthorized hub/switch additions.

#### B. Traffic Segmentation
*   **Student Isolation:** Client isolation enabled on WAPs to prevent peer-to-peer attacks within the student VLAN.
*   **Management Plane:** The Management VLAN (999) is an Out-of-Band (OOB) network, inaccessible from any user VLAN.

#### C. Wireless Strategy (Wi-Fi 6/6E)
*   **Floor 2 (CSE):** High-density deployment. 1 AP per 30-40 students to mitigate airtime contention.
*   **Floor 3 (ECE):** Medium-density deployment. 1 AP per 20-30 students.
*   **Sourcing:** All APs connect via PoE+ ports on the access switches, tagging traffic into the respective Student/Staff VLANs.

### 6. Summary of Redundancy Flow
$\text{User Endpoint} \xrightarrow{\text{PoE+}} \text{VSF Stacked Access Switch} \xrightarrow{\text{LACP/M-LAG}} \text{VSX Core Pair} \xrightarrow{\text{Firewall}} \text{Internet/Campus Backbone}$

## Phase 3: Device Selection & BOM

Based on the provided campus network requirements and the approved high-availability topology, I have selected the following HPE Aruba Networking hardware.

### Hardware Selection Logic

**1. Core/Distribution Layer (MDF - Floor 1):**
*   **Role:** VSX High Availability.
*   **Selection:** **Aruba CX 6400 Series**. The 6400 is a modular chassis switch designed specifically for the Core/Aggregation layer, providing the required VSX (Virtual Switching Extension) support to ensure a zero-single-point-of-failure architecture. It handles the high-speed routing and inter-VLAN traffic for the entire building.

**2. Access Layer (IDFs - Floor 1, 2, 3):**
*   **Role:** PoE+ delivery, VSF Stacking, and 10G Uplinks.
*   **Selection:** **Aruba CX 6300M Series**. 
    *   **Why CX 6300M over 6200?** The 6300M supports higher-speed uplinks (SFP56 up to 50G) and more robust VSF stacking capabilities, which are essential for the high-density requirements of the CSE and ECE floors. 
    *   **Port Density:** 48-port models were selected to meet the endpoint counts.
    *   **PoE Budget:** Class 4 (PoE+) and Class 8 (Smart Rate) options are available to support Wi-Fi 6/6E APs, VoIP phones, and cameras.

### Wireless AP Calculation
Based on the requirement of ~25-30 users per AP:
*   **Floor 2 (CSE):** 370 users $\div$ 30 $\approx$ **13 APs**.
*   **Floor 3 (ECE):** 216 users $\div$ 30 $\approx$ **8 APs**.
*   **Floor 1 (Admin):** 31 users $\div$ 30 $\approx$ **2 APs**.
*   **Total Building APs:** **23 APs**.

### Technical Summary of Implementation
*   **Core Redundancy:** The two CX 6400 switches will be configured in **VSX**, providing a single logical gateway with independent control planes.
*   **Access Redundancy:** Floor 2 and 3 switches are grouped in **VSF stacks**, simplifying management into a single virtual switch per floor.
*   **Uplinks:** Each access stack connects to the Core via **LACP/M-LAG bundles** using $2 \times 10\text{GbE}$ SFP+ links, ensuring 20Gbps aggregate throughput and link redundancy.
*   **Power:** CX 6300M switches provide sufficient PoE budget for the calculated APs, VoIP phones, and security cameras.
