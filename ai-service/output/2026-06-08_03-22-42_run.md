## Phase 2: Network Topology

As a Senior Network Topology Architect, I have designed the following high-availability campus architecture. This design utilizes a **Three-Tier Hierarchical Model (Core $\rightarrow$ Distribution $\rightarrow$ Access)** to ensure deterministic traffic patterns and linear scalability.

---

### 1. Topology Overview
*   **Topology Type:** Hierarchical Star / Hub-and-Spoke.
*   **Core Layer:** High-speed collapsed core (Dual Chassis) located in the Main Academic Block Server Room.
*   **Distribution Layer:** One Distribution Block per building to aggregate floor traffic and perform inter-VLAN routing.
*   **Access Layer:** Floor-specific PoE+ switches providing end-user connectivity.

### 2. Layer Breakdown & Sizing

#### Building 1: Main Academic Block (Distribution Block 1)
*   **Floor 1 (Admin/Reception):** 1x 48-port PoE+ Access Switch. (Load: 46 devices).
*   **Floor 2 (Comp Sci):** 2x 48-port PoE+ Access Switches in a **VSF Stack**. (Load: 229 devices). *High-density wireless APs integrated here.*
*   **Floor 3 (Electronics):** 2x 48-port PoE+ Access Switches in a **VSF Stack**. (Load: 167 devices).

#### Building 2: Library & Research (Distribution Block 2)
*   **Floor 1 (Library):** 2x 48-port PoE+ Access Switches in a **VSF Stack**. (Load: 213 devices). *High-density wireless APs integrated here.*
*   **Floor 2 (Research):** 1x 48-port PoE+ Access Switch. (Load: 60 devices).

---

### 3. High Availability & Link Design

#### Logical Redundancy
*   **Core/Distribution:** **VSX (Virtual Switching Extension)** is implemented at the Core and Distribution layers. This allows two physical switches to operate as one logical entity, enabling an active-active data plane.
*   **Access Layer:** **VSF (Virtual Switching Framework)** is used to stack switches on high-density floors (Comp Sci, Electronics, Library), reducing the number of managed IP entities and providing link redundancy.
*   **Link Aggregation:** All uplinks utilize **LACP (802.3ad)**. Access switches connect to Distribution switches via 2x10Gbps SFP+ bundles (Multi-chassis Link Aggregation).

#### Link Speeds
*   **User to Access:** 1Gbps (Cat6a).
*   **Access to Distribution:** 20Gbps (2x10G LACP Bundle).
*   **Distribution to Core:** 40Gbps (4x10G LACP Bundle).

---

### 4. VLAN & Subnet Plan
Each department/function is isolated. Subnets are sized using a `/24` for high-density students and `/26` or `/27` for staff/admin to optimize broadcast domains while allowing 20% growth.

**QoS Marking Strategy (HPE/Aruba Standard):**
*   **VOIP:** EF (Expedited Forwarding / DSCP 46) $\rightarrow$ Priority Queue.
*   **IPTV:** AF41 (Assured Forwarding / DSCP 34) $\rightarrow$ High Priority.
*   **Admin/Staff:** AF21 (DSCP 18) $\rightarrow$ Medium Priority.
*   **Students:** Default (BE - Best Effort).

### 5. Redundancy & Failover Mechanisms

*   **Gateway Redundancy:** **VRRP (Virtual Router Redundancy Protocol)** is configured on the Distribution layer. If the primary distribution switch fails, the backup assumes the Gateway IP instantly.
*   **Loop Prevention:** **MSTP (Multiple Spanning Tree Protocol)** is used to prevent loops while allowing load balancing across multiple VLANs.
*   **Physical Path Redundancy:** Dual-homed fiber runs from each building Distribution block to two separate Core switches (Cross-connect).

### 6. Security Zoning & RBAC Logic

*   **Zone Isolation:** 
    *   **Student VLANs** are prohibited from accessing **Admin/Staff VLANs** via Hardware ACLs at the Distribution layer.
    *   **Management VLAN (999)** is only accessible via a secure jump-box or encrypted SSH from the Admin subnet.
*   **Sensitive Area Controls:**
    *   **Server Room/Finance:** Implementation of **802.1X Port-Based Authentication**. Any unauthorized device plugged into a Finance port is automatically shut down.
*   **RBAC Access Policies:**
    *   **Principal/Finance Head:** Granted "Elevated Access" ACLs allowing routing into the Server Room's database segment and Financial Application servers.
    *   **Students:** Restricted to HTTP/HTTPS, DNS, and specific Academic Portals; blocked from all internal infrastructure IPs.

## Phase 3: Device Selection & BOM

Based on the approved topology and the retrieved technical specifications from the HPE Aruba Networking datasheets, here is the detailed hardware design and Bill of Materials (BOM).

### Hardware Selection Logic

1.  **Core Layer (High Availability):** 
    *   **Selection:** **CX 6400 Series**. The topology requires a dual-chassis collapsed core with VSX support for active-active data plane redundancy and 40Gbps uplink capacity. The CX 6400 is the modular chassis option providing the necessary stability and high-density 10G/25G/100G slots to aggregate distribution blocks.

2.  **Distribution Layer (Building Aggregation):**
    *   **Selection:** **CX 6300M Series**. These switches support **VSX**, which is mandatory for the Distribution blocks to provide VRRP gateway redundancy and LACP bundles to the core. They offer the necessary 10G/25G/50G SFP56 uplinks to meet the 40Gbps aggregate requirement to the core.

3.  **Access Layer (Floor Connectivity):**
    *   **Selection:** **CX 6300M** (for high-density floors) and **CX 6200F** (for lower-density floors).
    *   **High Density (CS, Electronics, Library):** Using **CX 6300M** allows for **VSF stacking** (up to 10 members) and "Smart Rate" ports (1G/2.5G/5G) which are critical for high-density Wi-Fi 6 APs.
    *   **Standard Density (Admin, Research):** **CX 6200F** is a cost-effective choice that still supports **VSF stacking** and provides the 10G SFP+ uplinks required for the 20Gbps LACP bundle to the Distribution layer.

### Technical Summary & Implementation Notes
*   **PoE Strategy:** CX 6300M models were selected for high-density areas to provide **Class 8 PoE (up to 90W)**, ensuring that high-performance Wi-Fi 6 APs and IPTV devices have ample power without overloading the budget.
*   **Stacking & Redundancy:** 
    *   **VSX** is deployed at Core and Distribution layers for zero-packet-loss failover.
    *   **VSF** is deployed at the Access layer to reduce management overhead (one IP per stack) and provide link redundancy.
*   **Uplink Capacity:** All Access $\rightarrow$ Distribution links utilize 2x10Gbps SFP+ LACP bundles. Distribution $\rightarrow$ Core links utilize 4x10Gbps (or 2x25Gbps) bundles via SFP56/SFP28 ports to meet the 40Gbps requirement.
*   **Growth:** The use of 48-port switches and VSF stacking allows the organization to add new switches to existing stacks as the population grows by the expected 50-200 users without needing to redesign the IP schema.
