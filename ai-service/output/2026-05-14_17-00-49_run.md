### 2. Device & Connectivity Requirements
The topology must provide connectivity for the following device types per floor:
*   **End-User Devices:** Laptops (BYOD and Managed), Desktop Workstations.
*   **Peripherals:** Network Printers, VoIP Phones.
*   **Infrastructure:** IP Security Cameras (CCTV), Wireless Access Points (WAPs).
*   **Growth Factor:** The design must accommodate a growth margin of **200–500 additional users** without requiring a core hardware replacement.

### 3. Performance & Reliability Specifications
*   **Uptime:** High Availability (HA) is critical. Implement redundancy at the Core and Distribution layers to eliminate single points of failure.
*   **Throughput:** Ensure low latency for high-compute floors (CS, IS, AI/ML). Provide 10GbE uplinks between Access and Distribution switches.
*   **Wireless:** High-density Wi-Fi 6/6E deployment to handle the student concentrations on floors 3-6.

### 4. Security & Logical Segmentation
The network must be segmented using VLANs and Access Control Lists (ACLs) to isolate traffic. 
*   **Sensitive Areas:** Implement enhanced security/monitoring for the Library, Server Room, Examination Hall, and Finance Office.
*   **Role-Based Access Control (RBAC):** Define specific priority and access policies for:
    *   **Principal:** Unrestricted access to all administrative and academic data.
    *   **Finance Head:** Secure, isolated VLAN with encrypted access to financial records.
    *   **Exam Controller:** Temporary "lockdown" VLAN capabilities for examination periods to prevent unauthorized data leakage.
*   **General Segmentation:** Separate VLANs for Students, Staff, Admins, VoIP, and CCTV/IoT devices.

### 5. Physical Constraints & Assumptions
*   **Topology:** Use a Three-Tier Hierarchical Model (Core $\rightarrow$ Distribution $\rightarrow$ Access).
*   **Cabling:** Assume Cat6a for horizontal cabling and Single-Mode Fiber (SMF) for vertical risers between floors.
*   **MDF/IDF:** Place the Main Distribution Frame (MDF) on Floor 1 (Office) and Intermediate Distribution Frames (IDF) on every floor.

## Phase 2: Network Topology

### 1. Topology Overview
**Type:** Three-Tier Hierarchical Campus Design (Core $\rightarrow$ Distribution $\rightarrow$ Access).
The design utilizes a **Collapsed Core/Distribution block** specifically for the PJ Building, acting as the central hub for all 6 floors. To ensure maximum uptime and non-blocking throughput for high-compute floors (CS, IS, AI/ML), the architecture employs a **Virtual Switching Framework (VSF/VSX)** to treat multiple physical switches as a single logical entity.

---

### 2. Layer Breakdown
*   **Core/Distribution Layer (MDF - Floor 1):** 
    *   Two high-capacity Layer 3 chassis switches configured in **VSX (Virtual Switching Extension)**.
    *   Acts as the Default Gateway for all VLANs (SVI) and handles inter-VLAN routing.
*   **Access Layer (IDF - Floors 1-6):**
    *   **Floor 1 (Office):** 1x 48-port PoE+ Switch.
    *   **Floor 2 (Library):** 2x 48-port PoE+ Switches (Stacked via VSF).
    *   **Floor 3 (ECE):** 3x 48-port PoE+ Switches (Stacked via VSF).
    *   **Floor 4 (CS):** 3x 48-port PoE+ Switches (Stacked via VSF).
    *   **Floor 5 (IS):** 3x 48-port PoE+ Switches (Stacked via VSF).
    *   **Floor 6 (AI/ML):** 2x 48-port PoE+ Switches (Stacked via VSF).
    *   *Sizing includes a 20% buffer for printers, VoIP, CCTV, and the requested 500-user growth margin.*

---

### 3. High Availability & Performance
*   **Device Redundancy:** VSX at the Core prevents Split-Brain scenarios and allows for ISSU (In-Service Software Upgrades).
*   **Link Redundancy:** Multi-chassis Link Aggregation (M-LAG/LACP) is used for all uplinks. Each Access stack has two 10GbE fiber uplinks—one to Core-A and one to Core-B.
*   **QoS Policy:** 
    *   **Platinum (EF):** VoIP / Video Conferencing.
    *   **Gold (AF41):** Administrative/Finance Data.
    *   **Silver (AF21):** Academic/Student Data.
    *   **Bronze (BE):** Guest/General Internet.

---

### 6. Redundancy & Failover
*   **Layer 3 Gateway:** Virtual Gateway (VRRP/HSRP or VSX Active-Gateway) is implemented at the Core. If Core-A fails, Core-B assumes the Gateway IP in < 1 second.
*   **Loop Prevention:** Rapid Per-VLAN Spanning Tree (RPVST+) is enabled as a fail-safe, though LACP/VSX eliminates the need for STP blocking on primary paths.
*   **Security Isolation:** ACLs applied at the SVI level to restrict `VLAN 20-60` from accessing `VLAN 10` (Admin) and `VLAN 100` (Exam).

### Hardware Sizing Logic
*   **Core/Distribution:** The **CX 6400 series** is selected for the Collapsed Core to meet the requirement for high-capacity Layer 3 chassis switches supporting **VSX (Virtual Switching Extension)** for active-active redundancy and non-blocking 100GbE throughput.
*   **Access Layer:** The **CX 6300M series** is chosen for the access layer. These are modular, stackable switches supporting **VSF (Virtual Switching Framework)**. This allows the floor-based stacks to be managed as a single logical entity. 
    *   *Port Density:* Each 48-port switch provides a base for the user counts, with the total stack size per floor calculated to include the requested 20% buffer for peripherals (CCTV, VoIP, Printers) and the 500-user growth margin across the building.
*   **Wi-Fi APs:** Based on the target of **25-30 users per AP**, quantities were calculated by dividing the total user count per floor (Students + Staff + Admins) by 25.
