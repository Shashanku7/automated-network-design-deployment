## Phase 2: Network Topology

# Campus Network Topology Design: PG Block

## 1. Topology Overview
**Topology Type:** Hierarchical Three-Tier (Core $\rightarrow$ Distribution $\rightarrow$ Access)
The design utilizes a **Collapsed Core/Distribution model** specifically for the PG Block to minimize latency and cost, while maintaining a scalable "Pod" architecture. This allows the organization to add additional buildings (Pods) to the Core without disrupting existing operations.

## 2. Layer Breakdown & Mapping

### Core Layer (Campus Backbone)
*   **Function:** High-speed packet switching, inter-VLAN routing, and gateway for external connectivity.
*   **Deployment:** Dual redundant Core Switches deployed in a **VSX (Virtual Switching Extension)** or **VSS** cluster to provide a single logical entity.

### Distribution Layer (Building Block)
*   **Mapping:** The PG Block is assigned a dedicated Distribution Block consisting of two high-performance multilayer switches.
*   **Role:** Aggregates all floors, enforces ACLs between departments, and handles routing for the building's subnets.

### Access Layer (Floor-Level)
*   **Floor 1 (Office):** Low-density PoE+ switches. Focus on VoIP and Admin stability.
*   **Floor 2 (AI/ML):** High-density 1GbE/10GbE switches. Increased backplane capacity for large dataset transfers.
*   **Floor 3 (CS):** Maximum density PoE+ switches. High port count to support the 300+ student population and multiple WAPs.

## 3. High Availability & Link Design

### Redundancy Mechanisms
*   **Control Plane:** **VSX/VSS** at the Core and Distribution layers to eliminate Spanning Tree (STP) blocked ports and allow active-active forwarding.
*   **Data Plane:** **LACP (802.3ad)** Link Aggregation bundles for all uplinks.
*   **Gateway Redundancy:** **VRRP (Virtual Router Redundancy Protocol)** for first-hop redundancy for all VLANs.
*   **Physical:** Dual-homed uplinks from every Access switch to both Distribution switches.

### Link Specifications
| Link Segment | Speed | Configuration | Redundancy |
| :--- | :--- | :--- | :--- |
| Access $\rightarrow$ Dist | 10Gbps | 2 x 10G SFP+ LACP Bundle | Dual-Homed |
| Dist $\rightarrow$ Core | 40Gbps | 2 x 40G QSFP+ LACP Bundle | Mesh/Full-Dual |
| Server $\rightarrow$ Core | 10/25Gbps | Single-mode Fiber | Dual-Homed |

## Phase 3: Device Selection & BOM

Based on the approved hierarchical three-tier topology and the provided user counts for the PG Block, here is the detailed hardware sizing and Bill of Materials.

### Hardware Sizing Analysis

**1. Core & Distribution Layers**
*   **Core Role:** Requires high-speed switching (40Gbps uplinks) and VSX support. The **Aruba CX 6300M** series is selected for its 50G SFP56 capability and VSX support, ensuring the backbone can scale to the projected 200 additional users.
*   **Distribution Role:** Aggregates floor access switches. The **Aruba CX 6300M** is again used here to maintain a consistent high-performance fabric and 10G/25G/50G uplink flexibility.

**2. Access Layer (Port Density & PoE)**
*   **Floor 1 (Office):** 30 users (25 Staff, 5 Admin) + Phones/Cameras.
    *   *Requirement:* ~40 ports.
    *   *Selection:* **Aruba CX 6100 48G PoE** (JL675A) providing 48 ports of 1GbE and PoE+ for VoIP phones and cameras.
*   **Floor 2 (AI/ML):** 172 users (150 Students, 20 Staff, 2 Admin).
    *   *Requirement:* High density and potentially higher throughput for AI datasets.
    *   *Selection:* **2x Aruba CX 6200M 48G PoE** (R8Q70A). This provides 96 ports. Given the 172 users, we primarily assume high Wi-Fi density; however, for fixed workstations, 2x 48-port switches provide a strong baseline.
*   **Floor 3 (CS):** 343 users (300 Students, 40 Staff, 3 Admin).
    *   *Requirement:* Maximum density.
    *   *Selection:* **3x Aruba CX 6200M 48G PoE** (R8Q70A). This provides 144 ports for fixed devices and WAP aggregation.

**3. Wi-Fi Access Points (AP)**
*   *Calculation:* $\approx 25\text{--}30$ users per AP.
    *   **Floor 1:** $30 \text{ users} \rightarrow 2 \text{ APs}$ (Coverage/Redundancy).
    *   **Floor 2:** $172 \text{ users} / 25 \rightarrow 7 \text{ APs}$.
    *   **Floor 3:** $343 \text{ users} / 25 \rightarrow 14 \text{ APs}$.
