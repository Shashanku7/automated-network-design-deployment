# Engineering Report: Campus Network Design - College Building

## 1. Project Overview
This report outlines the network architecture for a single-building college facility. The design focuses on a multi-departmental environment (CSE, AIML, ISE) spread across three floors, requiring a mix of high-density wireless connectivity and targeted wired infrastructure for staff and specialized peripherals.

**Network Type:** CAMPUS NETWORK
**Uptime Requirement:** Standard (Occasional brief outages acceptable)

---

## 2. Architecture Selection
**Model: 2-TIER (Collapsed Core + Access)**

**Justification:**
The project consists of a single building with approximately 1,017 total users and a limited number of wired endpoints. A 3-tier architecture is unnecessary as there are no separate distribution buildings and the endpoint count per floor does not justify a dedicated distribution layer. 

A Collapsed Core model using **Aruba OS-CX** switches provides the optimal balance between performance and simplicity. By consolidating the Core and Distribution roles, we reduce latency and management overhead while maintaining high availability through Virtual Switching Extension (VSX) at the core.

---

## 3. Capacity Calculations

### Departmental Analysis

| Department | Floor | Total Users (S+S+A) | AP Count (Users/25) | Fixed Endpoints (VoIP+IPTV+Prnt) | Staff Wired PCs | Total Endpoints | Growth (1.2x) | Req. Access Ports | Switch Count (48p) |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| **CSE** | 1 | 437 | 18 | 9 | 10 | 37 | 45 | 45 | 1 |

| **ISE** | 3 | 379 | 16 | 7 | 10 | 23 | 28 | 28 | 1 |
| **Total** | | **1,017** | **43** | **23** | **35** | **91** | **111** | **111** | **3** |

**Calculations Detail:**
- **AP Count:** $\lceil \text{Total Users} / 25 \rceil$
- **Total Endpoints:** $\text{VoIP} + \text{IPTV} + \text{Printers} + \text{Staff Wired PCs} + \text{APs}$
- **Growth Capacity:** $\text{Total Endpoints} \times 1.2$
- **Switch Sizing:** All access layers will utilize 48-port PoE switches to accommodate growth and AP power requirements.

---

## 4. Topology Design

### 4.1 Executive Summary
The design utilizes a collapsed core architecture with a VSX-enabled core pair and VSF-stacked access switches on each floor. This ensures a loop-free environment with active-active forwarding.

### 4.2 Core Layer Design
- **Role:** Layer 3 Gateway, Inter-VLAN routing, and Firewall connectivity.
- **Deployment:** A pair of high-performance Aruba CX switches configured in **VSX (Virtual Switching Extension)**.
- **Gateway:** VSX Active Gateway is used to provide a single virtual IP/MAC for all VLANs, eliminating the need for VRRP.

### 4.3 Access Layer Design
- **Deployment:** One stack of Aruba CX switches per floor.
- **Stacking:** **VSF (Virtual Switching Framework)** is used to manage the floor switches as a single logical entity.
- **Connectivity:** Each access stack is dual-homed to the Core VSX pair using **LACP bundles (Multi-Chassis LAG)**.

### 4.4 Wireless Infrastructure
- **AP Deployment:** High-density Wi-Fi 6E/7 APs distributed across floors.
- **Strategy:** 43 APs total. Staff requiring dual-connectivity (Ethernet + Wi-Fi) will be provisioned with both a physical port and WPA3-Enterprise wireless credentials.
- **Management:** Centralized management via Aruba Central.

### 4.5 Redundancy & Security Design
- **Redundancy:** Active-Active data plane via VSX Core and VSF Access.
- **L2 Security:** DHCP Snooping, ARP Protection, and Loop Protection enabled on all access ports.
- **Access Control:** 802.1X authentication for all staff/student ports; Dynamic VLAN assignment via RADIUS.

## 8. High Availability Summary
- **Core Layer:** Aruba CX VSX pair. Active-Active forwarding using **Active Gateway**.
- **Access Layer:** Aruba CX VSF stacks.
- **Uplinks:** All access stacks are dual-homed to the core using **Multi-Chassis LAG (MCLAG)** to eliminate Spanning Tree blocking and provide instantaneous failover.
- **Gateway:** No VRRP used; VSX Active Gateway provides superior convergence and simplified IP management.

This report provides a mathematically validated Bill of Materials (BOM) and architectural design for a single-building college facility. The design supports a high-density user environment (students, faculty, and admins) with a specific focus on dual-connectivity for staff workstations and high-bandwidth support for IPTV and VoIP services.

**Uptime Requirement:** Standard (Occasional brief outages acceptable).
**Architecture:** 2-Tier Collapsed Core (Core/Aggregation $\rightarrow$ Access).
